Introduction

What is Kloudbit?

Kloudbit is a serverless platform that helps you build robust applications without the hassle of back-end coding and server management. Instead, you can focus on making great web and mobile applications.

Getting Started

Kloudbit applications are served by an automatically generated RESTful API. To create your application, start by defining API resources that represent your business objects.

1. Create an Application

Begin simply by creating an application.

2. User Resource

All Kloudbit applications start with a User resource by default.

3. Add a Resource

Add a resource by defining resource and data parameters of various types (string, number, boolean, array and object).

Once saved, Kloudbit will automatically generate POST, GET, PUT and DELETE HTTP endpoints.

4. Try It!

Now, your Kloudbit application is ready to accept API calls.

User Objects

In most cases, users are the first objects you'll create for any client application. With Kloudbit, this is done by making an API call to http://app.kloudbit.com/v1/users. You'll need your KLOUDBIT TOKEN, which you can find in the dashboard.

Request:

POST /v1/users HTTP/1.1
Host: app.kloudbit.com
Authorization: Basic [KLOUDBIT_TOKEN]
Content-type: application/json

{"username":"johndoe@example.com","password":"passwd"}

{...}

Authenticating Users

Users in your application must be authenticated (via Oauth2) before interacting with any of your application resources. You can authenticate users by making an API call to http://app.kloudbit.com/v1/oauth2/token.

Request:

POST /v1/oauth2/token HTTP/1.1
Host: app.kloudbit.com
Authorization: Basic [KLOUDBIT_TOKEN]
Content-type: application/json

{"username":"johndoe@example.com","password":"passwd"}

Response:
{...}

This will return a bearer token that you can use to make API calls for any other resource in your Kloudbit API.
